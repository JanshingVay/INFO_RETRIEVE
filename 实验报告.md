
# 信息检索系统实验报告

## 一、引言
### 1.1 项目概述
本项目实现了一个功能完整的中文信息检索系统，包含高性能异步爬虫、双检索算法（TF-IDF向量空间模型 + BM25概率模型）、跨模态多媒体检索（90+语言支持）、相关反馈优化、人工评价及可视化模块。系统满足作业的全部要求，且在多项功能上有创新和优化。

### 1.2 环境与社会可持续发展考量
- **数据源选择**：选用国内主流科技媒体（IT之家、36氪、新华网等），均为正规新闻站点，无侵权内容
- **本地存储**：数据全部保存在本地，不向第三方服务器上传，保护隐私
- **去重优化**：通过URL去重和标题相似度去重，减少重复内容，降低存储和计算开销
- **异步架构**：采用asyncio+aiohttp异步爬虫，大幅减少网络等待时间，提高资源利用效率
- **模型懒加载**：CLIP等大模型首次使用时才加载，后续复用，减少内存占用

## 二、系统架构与模块设计
### 2.1 整体架构
```
用户查询（命令行）
    ├─→ 文本预处理模块 → 分词 → 停用词过滤
    ├─→ 倒排索引模块 → TF-IDF统计
    ├─→ 检索算法模块（TF-IDF/VSM 或 BM25）
    ├─→ 相关反馈模块 → Rocchio查询扩展/文档boost
    ├─→ 可视化模块 → Precision对比、P-R曲线
    └─→ 多模态模块 → Jina CLIP v2 → 文本→图片/视频
```

### 2.2 各模块设计思路

#### 2.2.1 数据采集模块（`async_crawler.py`）
**设计思路**：采用asyncio+aiohttp异步并发架构，避免同步阻塞，提升爬取效率。

**核心功能**：
- 10个数据源配置（IT之家、36氪、新华网、人民网、凤凰网、网易、新浪、环球、澎湃、和讯）
- 20个并发连接，自动User-Agent轮换，避免反爬
- 自适应延迟和指数退避重试，提高稳定性
- 50-100篇/秒爬取速度（对比同步爬虫1-2篇/秒）
- 自动去重（URL、标题相似度≥0.85）
- 目标文档数：500-700篇

#### 2.2.2 数据清洗模块（`data_cleaner.py`）
**设计思路**：清洗HTML标签、去除模板内容、去重，保证数据质量。

**核心功能**：
- HTML标签移除
- 版权声明、广告链接等模板内容过滤
- URL去重
- 标题相似度去重（SequenceMatcher≥0.85）
- 最小内容长度过滤（≥100字符）

#### 2.2.3 文本预处理模块（`preprocessor.py`）
**设计思路**：使用jieba分词，配合停用词表，提取有效关键词。

**核心功能**：
- jieba中文分词
- 内置中文停用词表（600+常用词）
- 数字过滤
- 单字符非中文字词过滤

#### 2.2.4 倒排索引模块（`inverted_index.py`）
**设计思路**：构建词到文档的倒排索引，计算TF-IDF统计，支持保存/加载。

**核心功能**：
- 倒排索引存储（词 → {doc_id: count}）
- 文档长度统计
- IDF计算（log(N/(df+1))+1）
- 索引JSON序列化

#### 2.2.5 TF-IDF向量空间模型（`vsm.py`）
**设计思路**：将查询和文档表示为向量，计算余弦相似度排序。

**核心功能**：
- TF-IDF向量计算
- 余弦相似度（稀疏向量高效计算）
- 相关反馈优化（Rocchio查询扩展）
- 文档boost（根据历史评分调整）

#### 2.2.6 BM25概率模型（`bm25.py`）
**设计思路**：BM25是比TF-IDF更优的概率检索模型，对词频饱和和文档长度进行归一化。

**核心功能**：
- BM25公式实现（k1=1.5, b=0.75）
- 词频饱和处理（避免高频词权重过大）
- 文档长度归一化（消除长文档天然优势）
- 概率IDF平滑（避免出现负值）
- 支持相关反馈优化

#### 2.2.7 跨模态多媒体检索（`multimodal_retrieval.py`）
**设计思路**：使用Jina CLIP v2，将文本、图片、视频映射到统一语义空间，支持90+语言查询。

**核心功能**：
- Sentence-Transformers封装（Jina CLIP v2）
- 图片编码：PIL → CLIP视觉特征
- 视频编码：OpenCV抽帧（1fps）→ 逐帧编码 → 均值池化
- 余弦相似度检索
- 模型懒加载单例（只在首次使用时加载，后续复用，避免重复加载）

#### 2.2.8 相关反馈模块（`relevance_feedback.py`）
**设计思路**：用户对结果评分后，优化下次查询，形成反馈闭环。

**核心功能**：
- 评分记录（1-5分）
- Rocchio查询扩展（从高评分文档中提取高IDF词）
- 文档boost（根据历史评分调整文档权重）
- 数据持久化（JSON）

#### 2.2.9 人工评价模块（`evaluator.py`）
**设计思路**：用户交互式评价结果，计算Precision@10、Recall等指标。

**核心功能**：
- 预设评价查询（10个科技相关主题）
- 交互式评分界面
- 反馈前后Precision对比
- 统计信息展示

#### 2.2.10 可视化模块（`visualization.py`）
**设计思路**：生成学术级图表，直观展示算法性能。

**核心功能**：
- Precision@10对比柱状图（TF-IDF vs BM25）
- Precision-Recall曲线
- 算法性能雷达图
- 索引统计图表
- 爬虫性能对比图

#### 2.2.11 主界面（`main.py`）
**设计思路**：命令行交互式菜单，整合所有模块，提供友好操作体验。

**核心功能**：
- 10个功能选项（爬取、索引、TF-IDF搜索、BM25搜索、算法对比、多模态、交互式查询、评价、可视化、状态）
- 自动检测是否需要爬取/索引
- 实时反馈提示

## 四、算法对比
### 4.1 TF-IDF vs BM25 对比
| 维度 | TF-IDF | BM25 |
|------|--------|------|
| 模型类型 | 向量空间模型 | 概率模型 |
| 词频处理 | 线性TF | 饱和TF（避免高频词权重过大） |
| 文档长度归一化 | Cosine归一化 | 显式长度归一化因子（b=0.75） |
| IDF公式 | log(N/(df+1))+1 | log((N-df+0.5)/(df+0.5))+1 |
| 参数 | 无可调参数 | k1=1.5（词频饱和）, b=0.75（长度归一化） |
| 性能 | 良好 | 通常更优（学术标准） |
